Sentiment Analysis

  • Tech Stack: Python (version 3.7+), Pandas, NumPy, Scikit-learn, SVM, KNN, DT, XGBoost, Random Forest, Logistic Regression
  • Github URL: Project Link

In my recent project, I undertook the challenge of analyzing 1.6 million tweets to perform sentiment analysis and feature engineering. This comprehensive process began with data cleaning and preprocessing, where I employed regular expressions (regex) to meticulously remove unuseful data such as emojis, URLs, and other irrelevant characters, ensuring the dataset was pristine for analysis. The dataset contained six columns, including the following: 'target,' indicating the polarity of the tweet (0 = negative, 4 = positive); 'ids,' representing the unique identifier for each tweet (e.g., 2087); 'date,' showing the timestamp of the tweet; 'flag,' indicating the query or NO_QUERY if no query was present; 'user,' denoting the user who posted the tweet; and 'text,' comprising the tweet's content. I then focused on feature engineering, extracting critical features such as the maximum and minimum lengths of tweets, among other relevant metrics, to enhance the model's predictive capabilities. I trained various machine learning models, carefully tuning their parameters to accurately classify the sentiments expressed in the tweets. I utilized advanced algorithms to optimize the model's performance, ensuring robust and reliable predictions. To facilitate seamless future use, I saved these trained models, allowing for efficient deployment and scalability.
